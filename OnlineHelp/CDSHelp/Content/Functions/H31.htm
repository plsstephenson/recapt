Create declaration for inbound item at post

You use the Create declaration for inbound item at post function for entering data into an electronic declaration form for inbound mail items. You can use it to create a new electronic declaration for an inbound mail item, which you can also share with your local Customs authority, or to edit an existing declaration received via ITMATT.

You cannot send an inbound item to a destination post, as the transit scenario is not covered by CDS.

To create or edit an inbound mail item declaration

  1. Enter the Item id. If you have a barcode scanner connected to your workstation, you can scan the S10 label (i.e. international mail identifier barcode) attached to the mail item.

  2. If the same mail ID already exists in CDS, you can update it, or create a new entry with the same international mail ID.

    To update an item, click Select next to the item you wish to edit, or New to create a new entry in the CDS database.

    If data is available from the Kiosk, click Fill from anonymous declaration, then enter the anonymous declaration ID when prompted. The system populates the fields with information. Otherwise, if data is not available, proceed to fill in the fields.

  3. Complete or review the general information:

    Field Description
    Destination post The postal operator as listed in the CDS reference dataset (e.g. CHA stands for Swiss Post).
    Mail class The type of mail: EMS, Letter or Parcel.
    Posting date The date and time the mail item was posted.
    Local ID (optional) A local identifier associated with the mail item used by some postal operators for domestic distribution. This is particularly useful in differentiating between duplicate mail items.
    Mail state

    (optional) Identifies the state of the mail item according to its EMSEVT tracking state.

    The following outbound states are allowed:

    • A - received from customer (equivalent to EMSEVTv1 EMA)
    • B - arrived in outbound OOE (EMSEVTv1 EMB)
    • B1 - arrived in outbound customs (EMSEVTv3 EXA)
    • B2 - marked as blocked in OOE (state provided to reflect watchdog functionality)
    • B3 - marked with warning in OOE (state provided to reflect watchdog functionality)
    • C - departed OOE (equivalent to EMSEVTv1 EMC)

    and the following inbound states are allowed:

    • D - arrived at IOE
    • E - arrived at customs
    • F - departed IOE

  5. Complete the declaration by selecting one of the following actions from New action and clicking Store (note that only the options relating to the current configuration are available):
    • 1 (Create/update draft) saves a draft declaration for later editing without sharing it with another organization
    • 2 (Store final (automatic transfer)) stores the declaration; it will be automatically transferred to local Customs or to the destination post if the administrator user of the export post configured rules for this automatic forwarding
    • 6 (Send to customs) sends the declaration to local Customs for export analysis; local Customs can then cancel the export or clear its way, or provide any other response that may make sense within the local context
    • 7 (Send to dest post) sends the declaration to the destination post
    • 8 (Delete) deletes a previously created declaration; this function is typically used to delete a previously stored draft declaration that is no longer relevant
  6. Click Store or Store & print CN23. If you choose to print the CN23, see the following sections for more details of available formats.

Printing a Declaration Form

By default, CDS produces an A4 CN23 paper print-out of the details entered into the electronic form.

Printing a Declaration Form on A5 paper and reducing the size further

Before printing, check that your printer is properly configured for your desired page orientation: portrait or landscape.

You can print the CN23 form on A5 (148 x 210mm) paper. This is the smallest print-out format possible for the CN23 form without sacrificing readability. If your printer, however, has only A4-sized paper loaded and you selected to print the CN23 form on A5, the form will be printed on the center of the page.

Should you wish to scale down the form size further for packaging purposes, you can print it on A6 paper. Select your print page setup to A6 (105 x 148mm) format then set the size option to Shrink oversized pages. Alternatively, you can fold the A5 CN23 form to fit on a small package.
