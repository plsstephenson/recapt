Manage organization preferences

The Manage organization preferences function allows you to set the organizationâ€™s default preferences.

The preferences you can update are summarized in the table below.

Field Description
Associated customs organization (Postal organizations only) Name of the Customs organization associated with the postal operator.
Currency

Default currency code used by the organization.
Language Default user interface language of the operators. This parameter can be overridden at user level using Manage my preferences (see Manage my preferences). For example, an operator can have a default language of English, but individual users can set their own language to French.
Security token (for accessing CDS.API) This token is generated by clicking on New. The token generated can then be used by client software connecting to CDS.API to access the CDS Web Services.
Date format Used for setting the date format used to display dates. The two available options are dd/mm/yyyy and mm/dd/yyyy.
Time zone Used for setting the default time zone of the organization.
Retentions (in days) for declarations and responses before archiving Used for setting the number of days that declarations and responses are kept in the main operational database before sending them to the archive database.
Generate ITMATT when item departs OOE If the Generate ITMATT when item departs OOE box is:
  • unchecked, CDS generates an ITMATT message as soon as a declaration is in a valid transfer state, and an EDI exchange has been defined with the destination post of a given mail item
  • checked, CDS only generates an ITMATT message once the mail item is associated with a C-scan (i.e. departure from office of exchange)

You can associate a mail item with a C-scan using the application interface or the CDS.API web services server. Typically, IPS performs a C-scan using the operational screen in IPS, which then synchronizes with CDS in the background (if IPS-CDS synchronization has been configured).

You can also opt to generate ITMATT when the item arrives at the OOE or when the item is posted.
Extra fields for declarations

Any number of additional fields can be added to declarations by listing them here, separated by commas. These fields will be added at the bottom of the Other section of the screen used for editing Customs declarations.

Although these fields are visible in the application, they are not yet included in EDI messages.
Required fields for outbound/inbound process You can set any declaration field to mandatory for inbound and/or outbound declarations by checking the check box next to it.

To set a new field as mandatory, you first need to store the newly declared field, and then it will appear in the list of declaration fields.
