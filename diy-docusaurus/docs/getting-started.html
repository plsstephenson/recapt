Getting started

Welcome to DIY! DIY is a cloud-based platform for assembling custom API flows simply, enabling data connectivity for your internal and external business needs.

Introduction

Using DIY, you can connect data sources -- such as web APIs and databases -- and expose data models, through secure, RESTful APIs. You can create a seamless flow from one API to the next, passing data along as you go. You can also manipulate the data before it reaches its destination using a graphical drag and drop interface. Once you have configured your flow, it appears to your customer as a single API, which they can access using state of the art API documentation.

DIY is available in English and French.

What you need

DIY uses a web interface and cloud hosting, so there is nothing for you to install. All you need to get started is a user account and access token. See DIY for administrators for more details.

How it works

You can assemble your custom APIs in minutes. All you need to do is:

  • specify your list of data sources for connection
  • give users access to your new API

DIY comes with out-of-the-box functionality providing everything you need to assemble simple APIs. If you want to make your APIs more complex, however, DIY also provides the tools you need to extend it -- see DIY for developers for an overview of how to add some extra configuration to your API and DIY for administrators for an explanation on how to create your own data models.

Create the list of data sources

  1. Open API > Datasources.
  2. Click .
  3. Select the data source connector type from the drop-down list and click Next. If the model you want is not available, create it using the steps in Declare new data models.
  4. Complete the details of the data source configuration either in the Form tab or the Editor tab (mandatory fields are indicated with a red asterisk). The fields available depend on the type of data source specified.
Connector typeDetails to specify
Database (mySQL, Oracle, etc.)Host server/port, user credentials and database name
REST webserviceList all the endpoints you need, including URL, method, parameters, etc.
SOAP webserviceURL and WSDL
Storage connectorStorage provider details, including host server/port, user credentials and root folder
  1. Click Done. The connection is tested and an error reported if connection is not successfully established. Repeat the steps from step 2 for each datasource you want to connect.

After you have created the data source, a new data model definition is created for you. You can see the details in API > Models.

Share the API

  1. Open API > API explorer. Datasources and models are presented in OpenAPI format.
  2. To share your API with customers and other users, create a new user account with the appropriate access control rules. See DIY for administrators.
  3. Once you have granted the appropriate access control rules, you can share the API explorer URL with your customers where they can view and try out your API endpoints.
