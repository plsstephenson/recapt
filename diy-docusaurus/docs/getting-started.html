Getting started

Welcome to DIY! DIY is a cloud-based platform for assembling custom API flows simply, enabling data connectivity for your internal and external business needs.

Introduction

Using DIY, you can connect data sources -- such as web APIs and databases -- and expose data models, through secure, RESTful APIs. You can create a seamless flow from one API to the next, passing data along as you go. You can even manipulate the data before it reaches its destination using a graphical drag and drop interface. Once you have configured your flow, it appears to your customer as a single API, which they can access using state of the art API documentation.

DIY is available in English and French.

How it works

Assembling custom APIs with DIY is simple! All you need to do is:

  • specify your list of data sources for connection
  • give users access to your new API

It really is that simple! Of course, you can always complicate things a little bit -- see Declare new data models and Manipulate data for an overview of how to add some extra configuration to your API. More details can be found in DIY for developers.

Create the list of data sources

  1. Open API > Datasources.
  2. Click the + icon.
  3. Select the data source connector type from the drop-down list and click Next.
  4. Complete the details of the data source configuration either in the Form tab or the Editor tab (mandatory fields are indicated with a red asterisk). The fields available depend on the type of data source specified.
Connector typeDetails to specify
Database (mySQL, Oracle, etc.)Host server/port, user credentials and database name
REST webserviceList all the endpoints you need, including URL, method, parameters, etc.
SOAP webserviceURL and WSDL
Storage connectorStorage provider details, including host server/port, user credentials and root folder
  1. Click Done. The connection is tested and an error reported if it is unsuccessful. Repeat the steps from step 2 for each datasource you want to connect.

Publish the API

  1. Open API > API explorer. Datasources and models are presented in OpenAPI format.
