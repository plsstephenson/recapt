Manage rules for customs responses

You use the Manage rules for customs responses function to configure:

  • automated response rules for outbound and inbound declarations sent to Customs by their local Post
  • the transfer of these responses to the sending Post, in the case of inbound declarations

To configure rules for automated responses to a local Post

  1. Click the Auto response rules tab. You can create one or more auto response rules for each type of Customs decision. CDS checks every declaration against your configuration and, when a rule matches, provides one of the following automatic responses:
    • Item must be presented
    • Item is cleared
    • Item is cleared after being checked
    • Charge raised, but Customs do not need to see it
    • Refer to other government department
    • Enforcement
    • Detention
    • Item is not allowed in the country
    • Other
    • Any other decision you may have defined with the Manage customs decisions function

    In a local CDS installation, it is possible to expand the list to include any other responses required in the country implementing the system. These additional decisions can also be automated following the same logic described in these sections.

  1. To create one or more auto response rules, click Edit next to the Customs decision.
  2. Enter the priority of the auto response rule. When it comes to applying the rules, rules with greater priority (a higher number) will be applied first.
  3. For each auto response rule you want to add for this type of Customs decision, click Add auto response rules. The rules are listed as rows in the table. For each rule, click Edit next to it, then complete the following steps in the inbound or outbound mail section.
  4. Check the required rules from A to R and specify any corresponding values.
  5. Enter your combination of criteria in the Logical expression field and use the logical operators AND and OR in combination with brackets (if necessary).
  6. Finally, you can randomize responses by setting a value between 0 and 100 in the Applies to a percentage of items of textbox.

To configure rules for automatically transferring responses to the origin Post for inbound mail:

  1. Select the Origin post transfer rules tab. The response decision code depends on the rule defined for automatic responses to the local Post.
  2. Configure the rules for additionally creating a response to the sending Post if certain criteria are met.
  3. Enter your combination of criteria in the Logical expression field and use the logical operators AND and OR in combination with brackets (if necessary).

To view auto response rules:

Use the Inbound Selectivity Rules Dashboard function to view all automated response rules defined in the system. You can choose to view all rules associated with a decision name or all rules for all decisions and then export the report.

  1. Click the Inbound Selectivity Rules Dashboard button from your Home page. By default, the system displays your user ID in the CurrentUser field.
  1. In the DecisionName field, enter a specific decision name or leave it blank to view all rules for all decisions.
  2. Click Execute. CDS displays the results. Details of the rule associated with the decision are displayed.
  3. Click Export to export the report to a comma separated file.
