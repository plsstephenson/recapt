Enterprise user

Imagine you are working in a company with a set of different systems which don't talk to each other. You want to set up a link between them:

  • CRM system

Perhaps you also have a proprietary system with its own database? This is where DIY's data model definition comes in handy. You can set it up however you like, then declare it in a data source along with your other systems.
