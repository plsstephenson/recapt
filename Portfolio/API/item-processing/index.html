    Introduction

    The XXX API exposes xxx functions.

    This API reference explains each of the API endpoints, with example requests and responses in the right-hand column of the page.

    Authorization

    To use the XXX API, you need to include a token with each HTTP call (a GUID that is associated with one or more API operations).

    Response codes

    Code Meaning
    200 Success
    400 Bad Request -- data in the request body is invalid
    401 Unauthorized -- authorization failed
    500 Internal Server Error -- an exception error occurred

    If an error or a code other than HTTP 200 is returned, see the response body for details (the format of an error response body is shown in the column on the right).

    Response body format

    
    {
      Errors: [
        "Error message 1", "Error message 2"...
      ]
    }

    XXX list

    Retrieves a XXX list.

    HTTP request

    GET http://[host name]/APIService/ListService.svc/rest/dnl?agentId={agentid}&token={token}

    where {agentId} is the agent code and {token} is the required GUID for authorization. If agentid is null, the response contains a delivery list for each agent who has mail items to deliver.

    Example response (200)

    {
    "DeliveryAgentCode":"String content",
    "Mailitems":[
      {
        "Identifier":"String content",
        "LocalIdentifier":"String content",
        "MailItemDeliveryInformation":{
          "COD":{
            "Amount":12678967.543233,
            "Currency":"String content"
          },
          "DeliveryAddress":{
            "Address":"String content",
            "City":"String content",
            "Forename":"String content",
            "Name":"String content",
            "PhoneNo":"String content",
            "Postcode":"String content"
          }
        }
      }
    ]
  }

    Success

    If successful, this method returns an HTTP 200 response with items grouped by agent. If agent is null, the structure is repeated for each agent listed.
